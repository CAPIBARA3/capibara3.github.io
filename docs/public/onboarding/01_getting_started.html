01 Getting Started#

Goal#

Once you have submitted the application and received an invitation to join the CAPIBARA Collaboration per email, you will need to complete the following steps to become a full member of the collaboration.

Prerequisites#

  • Have submitted the application form

  • Fulfill the membership requirements (see membership agreement)

  • Have received invitation to join the collaboration via email from the official collaboration email.

Steps#

1. Introduction#

Explain context.

2. Setup#

Before starting this process you will need to have created a GitHub account. If you have one, you just need to remember the login credentials (for you to log in). If not you can find a very detailed guide here. GitHub is our main information hub all of our codes, documentation and files are stored in our organisation (github.com/capibara3). Decision-making also takes place there through asynchronous communication within issues, pull requests, and discussions. You will learn the “GitHub workflow” once you reach the next tutorials.

3. Execution#

In order to finalise the joining process, you need to enter the CAPIBARA GitHub organisation. For that matter, you will need to submit the following information by replying to the email with the invitation to join the collaboration.

  1. GitHub username: so that we can invite you to join the organisation on GitHub

  2. For our team website (capibara3.github.io/team) you may want to provide the following information:

  • Full Name

  • Short statement

  • Description

  • Profile Picture

  • Links (LinkedIn, YouTube, website, email, get inspired by other profiles!)

  • Short presentation video (30 sec max)

Providing this data is optional, and you may also request not to have a public profile or to delete your data anytime. By sending the data, you agree to it being treated as outlined here. It will be hosted on GitHub Pages and be publicly available on the CAPIBARA Collaboration’s website.

After sharing this information, you may proceed to read the most important documentation for best practices:

  • Code of Conduct

  • Contributing guidelines

  • Governance overview

  • Authorship

  • Licensing

  • Communication

  • …

4. Verification#

Next Steps#

Once you have joined the CAPIBARA GitHub organisation, you can start contributing! Plus, you also have access to all of our private files. Make sure to always comply with the code of conduct and contribution expectations, both collaboration-wide and repository-specific guidelines. You will learn about GitHub in a future tutorial. Now, let’s set up your computer: 02_setting_up_unix.md.
